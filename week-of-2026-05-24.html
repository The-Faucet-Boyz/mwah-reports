⭐ Highlights of the week
✉️

Portal emails — fully rebuilt, reliable, and visible

The biggest portal infrastructure story of the year. Portal emails migrated off Gmail OAuth — which had broken twice in two weeks when the refresh token expired — onto Resend, a dedicated transactional mail service with no expiry risk. At the same time, every portal email that sends now writes an audit log entry: delivery time, success or failure, which template, which buyer. A new Email Health tab (the 8th in Portal Mgmt) shows sent/failed counts, a 7-day chart, per-template success rates, and a one-click Retry button when something goes wrong. On the buyer side, the emails themselves got a full formatting pass so Gmail, Outlook, and mobile all render them cleanly. New lifecycle emails went live too: buyers now hear from us when their order is cancelled (with reason + any credit note), when a license correction gets resolved, and when a rep grants them access to an additional store.

Buyer portal
🔑

Managers can now fix PO mistakes on any order — even locked ones

If a rep confirmed an order with the wrong store, a bad delivery date, or an incorrect line-item price, fixing it used to require manual database surgery. Managers and admins can now click a pencil icon on any PO — regardless of status — to reassign the store, change the delivery date, or correct a line-item quantity, price, or sample flag. A required reason field feeds an edit-history banner on the PO so the full diff is always visible. This came directly out of a real case where a rep accidentally placed an order for the wrong Bakeree location.

⚙️ Behind the scenes
  • Portal emails migrated from Gmail OAuth to Resend — eliminates the refresh-token expiry that broke buyer emails twice in two weeks.
  • Every transactional portal email now writes a structured audit log entry; failures surface in Sentry within seconds; a weekly canary send catches a dead configuration before any buyer notices.
  • Intercom Identity Verification enabled on the buyer portal — buyers can no longer spoof another buyer's chat history via browser devtools.
  • GrowFlow API fixed: an unused field in the orders query was triggering a server error on GrowFlow's end, silently breaking the daily 6am sync and the manual Live Sync button.
  • Reconciliation hardened: license mismatches now block PO transfer (was silently passing through); non-blocking flags get their own bulk-action section; unresolved decision counts dedupe correctly across multiple sync runs.
  • CHANGELOG array extracted to its own file — trimmed dashboard.js by ~27% and stopped the lint-ceiling ratchet that had bumped the file-size cap eight times.
  • Self-review CI workflow added: every non-draft PR now requires a structured attestation line, enforced by GitHub Actions at merge time.
  • Fulfillment page infinite-load bug fixed — the table now finishes rendering after an app update without needing a hard refresh.
